                \section{Da Natureza deste Documento}
                \paragraph{}
Este laudo objetiva quantificar e tipificar os erros, falhas e omissões,
levantados durante a execução do método de análise de requisitos
\textit{PBR} na documentação existente para o projeto em questão. Tais
falhas, omissões ou problemas deverão ser corrigidos para a correta
implementação do projeto. Este documento é elaborado com base nas
análises específicas das áreas de requisito -- analista de requisitos,
usuário e testador.

                \section{Resultado}
                \subsection{Análise Documental: defeitos encontrados}
        \subsection{Condições Gerais}
\begin{enumerate}
\item consistência de interfaces: não é possível compreender de que gui vai para
outra;
\item Disponibilidade de informações: parcialmente disponíveis. Não determina com quais sistemas deve se comunicar. Como
a comunicação deve ser feita?
\item As funções não estão todas especificadas - caso da palavra chave, que não possui requisito, domínio não é
especificado;
\item Interface entre funções: não são bem definidas e compatíveis. Dados desconexos entre
si;
\item o fluxo das atividades precisa ser deduzido, o que leva a
diferentes interpretações.
\end{enumerate}
